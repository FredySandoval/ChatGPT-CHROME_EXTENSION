## Reporting a Vulnerability

Please do not disclose security vulnerabilities in public GitHub issues, discussions, or pull requests.

To report a vulnerability privately:

1. Open the repository’s **Security and quality** tab.
2. Select **Report a vulnerability**.
3. Provide a description of the issue, affected versions, reproduction steps, potential impact, and any suggested fix.

For security-related questions that do not reveal a vulnerability, exploit, credentials, private data, or other sensitive information, you may open a regular GitHub issue.

If private vulnerability reporting is unavailable, open an issue containing only a request for a private communication channel. Do not include technical vulnerability details in that issue.
